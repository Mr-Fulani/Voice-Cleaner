# Информация о Docker контейнерах

## Почему не видно запущенных контейнеров?

При запуске используется флаг `--rm`, который **автоматически удаляет контейнер после завершения**:

```bash
docker run --rm ...
```

Это нормальное поведение! Контейнер:
1. Запускается
2. Обрабатывает файлы
3. Завершается
4. **Автоматически удаляется** (флаг `--rm`)

## Как проверить, что Docker работает?

### Вариант 1: Проверить во время выполнения

Откройте второй терминал и выполните:
```bash
docker ps
```

Вы увидите контейнер, пока он обрабатывает файлы.

### Вариант 2: Запустить без --rm (для отладки)

```bash
docker run \
  -v $(pwd)/fixtures:/app/fixtures \
  -v $(pwd)/output:/app/output \
  voice-cleaner \
  --preset max_voice
```

Затем проверьте:
```bash
docker ps -a | grep voice-cleaner
```

### Вариант 3: Проверить логи

```bash
docker logs <container_id>
```

## Рекомендуемый способ запуска

Используйте `--rm` (как в README) - это правильный подход:
- Контейнеры не накапливаются
- Диск не засоряется
- Всё работает автоматически

## Проверка работы Docker

Если хотите убедиться, что Docker работает, запустите:

```bash
docker run --rm voice-cleaner --help
```

Вы должны увидеть справку по использованию.
